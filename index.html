Azure AD Setup Guide
Microsoft Entra ID · App Registration
Obiettivo

Questa guida spiega come registrare un'applicazione su Azure Active Directory (Microsoft Entra ID) per abilitare l'integrazione Outlook nella dashboard: invio email, creazione task, lettura bidirezionale dei task di flotta. Le permission OAuth2 richieste sono Mail.Send, Tasks.ReadWrite, User.Read.

1
Accedi al portale Azure

Vai su portal.azure.com con un account amministratore del tuo tenant Microsoft 365 aziendale. Dal menu laterale seleziona Microsoft Entra ID (ex Azure Active Directory).

2
Registra una nuova applicazione

Vai su App registrations → New registration.
— Name: Maritime Fleet Dashboard
— Supported account types: Accounts in this organizational directory only (Single tenant)
— Redirect URI: https://micheledv74.github.io/maritime-dashboard/ (tipo: Single-page application)
Clicca Register.

3
Copia il Client ID e Tenant ID

Nella pagina Overview dell'app trovi:
— Application (client) ID: incollalo nella variabile AZURE_CLIENT_ID
— Directory (tenant) ID: incollalo nella variabile AZURE_TENANT_ID
del file index.html nella sezione const AZURE_CONFIG.

4
Aggiungi le API Permissions

Vai su API permissions → Add a permission → Microsoft Graph → Delegated permissions.
Aggiungi le seguenti permission:

User.Read Mail.Send Tasks.ReadWrite MailboxSettings.Read

Clicca Grant admin consent per il tuo tenant.

5
Configura Authentication (PKCE per SPA)

Vai su Authentication e verifica che sia abilitato:
— Access tokens ✓
— ID tokens ✓
Poiché è una SPA su GitHub Pages, usare il Authorization Code Flow con PKCE tramite la libreria @azure/msal-browser (inclusa via CDN al prossimo step).
Non creare mai un Client Secret per app pubbliche su browser.

6
Aggiorna index.html con le credenziali

Cerca nel file la sezione const AZURE_CONFIG e inserisci:
clientId: "IL_TUO_CLIENT_ID"
tenantId: "IL_TUO_TENANT_ID"
Il flusso OAuth2/MSAL sarà attivato al prossimo step di sviluppo. L'integrazione Outlook sarà operativa dopo il push su GitHub Pages.

7
Test dell'integrazione

Apri la dashboard su GitHub Pages → clicca "Connect Outlook" nella sezione To Do.
Effettua il login con il tuo account Microsoft 365 aziendale.
Verifica che appaia il nome utente nel badge in alto a destra e che i task di Outlook siano visibili nel pannello.
Se ricevi un errore AADSTS700054: verifica i Redirect URI nell'app Azure.
Se ricevi AADSTS65001: rifare il Grant Admin Consent al punto 4.

Prossimo Step

Al prossimo step integriamo MSAL.js (libreria Microsoft Authentication Library) direttamente nel file per abilitare il login OAuth2, la lettura bidirezionale dei task Outlook e l'invio email via Microsoft Graph API. Forniscimi il tuo Client ID e Tenant ID dopo la registrazione e aggiungiamo il codice.
